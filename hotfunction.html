/token-creator/solana — 权限设置区域
🔴 Critical · P10安全不可逆操作置信度 9/10
问题描述

Token Creator 提供「撤销元数据修改权限」「撤销冻结权限」「撤销铸造权限」三个开关。这三项操作一旦执行，链上永久不可逆，直接影响 Token 的未来运营能力。文档仅一句话描述其功能，未在 UI 界面提供视觉警告、二次确认弹窗或后果说明。新用户误操作的风险极高，且无法回滚。

建议修复

为每个「撤销权限」开关添加橙/红色警告图标；在用户勾选后弹出确认模态框，用加粗红字注明「此操作不可逆，撤销后无法恢复任何权限」；要求用户输入 Token Symbol 进行二次确认（参考 GitHub 删除仓库的交互模式）。

修复提示
// 撤销权限二次确认弹窗示例 Modal.confirm({ title: '⚠️ 不可逆操作确认', content: `您即将撤销「铸造权限」，此操作链上永久生效，无法恢复。 请输入您的 Token Symbol「${symbol}」以确认：`, inputRequired: true, validateInput: (val) => val === symbol, danger: true, confirmText: '我已了解风险，确认撤销', })

Token Logo 上传缺少格式/大小校验与预览功能
/token-creator/solana — Logo 上传区域
🔴 Critical · P8表单验证用户体验置信度 8/10
问题描述

文档要求 Token Logo 为 1000×1000px，但未说明可接受的文件格式（JPG/PNG/WebP/SVG？）、最大文件体积、以及上传非正方形图片时如何处理。如果用户上传了格式错误或尺寸不合规的 Logo，Token 元数据中的 Logo 可能显示为损坏或比例失真——而这个问题在创建后才会被发现，且链上元数据已写入，修复需要额外操作。

建议修复

上传框需在客户端实时校验格式和尺寸；提供拖拽裁剪功能（至少限制为正方形）；显示上传预览；提示可接受的格式列表和最大尺寸；对非 1000×1000px 的图片给出警告但允许继续（而非报错阻断）。

修复提示
// Logo 上传校验逻辑 const ALLOWED_TYPES = ['image/png', 'image/jpeg', 'image/webp'] const MAX_SIZE_MB = 5 function validateLogo(file) { if (!ALLOWED_TYPES.includes(file.type)) throw '仅支持 PNG / JPG / WebP' if (file.size > MAX_SIZE_MB * 1024 * 1024) throw `文件不超过 ${MAX_SIZE_MB}MB` const img = new Image() img.onload = () => { if (img.width !== img.height) showWarning('建议使用正方形图片以获得最佳显示效果') if (img.width < 500) showWarning('建议分辨率不低于 500×500px') } img.src = URL.createObjectURL(file) }

Token 精度（Decimals）默认值与含义缺乏 inline 说明
/token-creator/solana — Decimals 输入区
🟠 High · P7用户体验新手友好置信度 8/10
问题描述

「Decimals（精度）」字段对大多数新用户而言是陌生概念。精度决定了 Token 最小交易单位——设为 0 表示不可分割（如 NFT），设为 9 是 Solana 标准（如 SOL），设为 6 则近似 USDC。一旦创建后无法修改，设置错误会导致 Token 在 DEX 上显示异常价格，或无法正常参与 AMM 定价。文档虽有说明但 UI 内没有任何 tooltip 或示例。

建议修复

在 Decimals 字段旁添加 ⓘ 图标；悬停时展示：「精度决定代币最小单位。推荐设置：9（与 SOL 一致）。设为 0 时代币不可分割，适合 NFT 类项目。创建后无法修改。」

接收方账户不存在时的 Rent 费用未提前告知
/multi-sender/solana — 费用确认步骤
🔴 Critical · P9Solana 机制费用透明置信度 9/10
问题描述

向从未持有该 SPL Token 的地址发送代币时，需要为接收方创建 Token Account，此时发送方须额外支付约 0.002039 SOL 的 Rent 费用（每个地址）。如果向 500 个新地址空投，仅 Rent 费就需要额外 ~1 SOL。文档未在发送前告知此费用，用户在签名钱包弹窗时看到远超预期的 SOL 扣款，极易引发恐慌和信任危机。

建议修复

在地址列表验证完成后，自动检测哪些地址尚未持有该 Token 的 Account；在确认页面显示「新账户数：X 个，需额外支付 Rent：~X.XX SOL」的明细警告；让用户对费用有明确预期。

修复提示
// 发送前预检 async function checkRecipientAccounts(addresses, tokenMint) { const newAccounts = [] for (const addr of addresses) { const ata = getAssociatedTokenAddress(tokenMint, new PublicKey(addr)) const info = await connection.getAccountInfo(ata) if (!info) newAccounts.push(addr) } const rentCost = newAccounts.length * 0.002039 // SOL showWarning(`${newAccounts.length} 个地址尚未持有此 Token 需额外支付 Rent：~${rentCost.toFixed(4)} SOL 此费用可在接收方关闭账户后取回`) return { newAccounts, rentCost } }

文档截图中私钥输入区域疑似为普通文本框（明文显示）。若恶意扩展、肩窥攻击者、截屏工具或屏幕录制软件捕获到私钥内容，将直接导致对应钱包资产被盗。同时，页面进入时没有醒目的安全警告横幅告知用户当前操作的敏感性。
