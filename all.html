Bundled Buy/Sell 工具页：用户私钥导入流程存在极高安全风险
slerf.tools/en-us/bundled-buy · /bundled-sell · /multi-address-bundle
🔴 Critical · P10 安全 私钥 置信度 9/10
问题描述

根据文档，「捆绑买入/卖出」「批量交易」「市值管理」等工具需要用户在页面表单中直接输入多个钱包的私钥（支持标准私钥和 Base58 格式）。文档虽然提到私钥在本地处理，但未在工具页面显著位置展示安全审计证明。私钥输入框如被 XSS 攻击或恶意扩展劫持，将造成资产直接损失。

建议修复

修复提示
在每个私钥输入框上方添加警告组件： ┌─────────────────────────────────────────┐ │ ⚠️ 安全提醒 │ │ 请勿在不信任的网络或设备上输入私钥 │ │ 本工具在浏览器本地执行，私钥不会上传服务器 │ │ 建议操作前验证当前域名为 slerf.tools │ │ [查看开源代码] [查看安全审计报告] │ └─────────────────────────────────────────┘ 同时在工具页面 URL 栏旁显示 HTTPS 锁定图标说明
